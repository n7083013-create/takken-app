# App Store 申請ガイド

## 📋 全体の流れ

```
1. Apple Developer Program 登録（$99/年）
   ↓
2. App Store Connect で新規アプリ作成
   ↓
3. EAS Build で iOS ビルド生成
   ↓
4. TestFlight でテスト
   ↓
5. ストア情報入力（スクショ・説明文等）
   ↓
6. App Review 提出
   ↓
7. 審査 (1〜3日)
   ↓
8. 公開
```

## 1️⃣ Apple Developer Program 登録

**URL**: https://developer.apple.com/programs/enroll/

### 費用・条件
- 費用: **$99/年**（約¥15,000・毎年更新必要）
- 必要情報:
  - Apple ID（二段階認証必須）
  - 身分証明書（パスポート・運転免許証）
  - クレジットカード
  - **個人** or **法人（合同会社カケル）** として登録可能

### 法人登録の注意
法人登録には **D-U-N-S Number** が必要（登記情報から無料取得可能）
https://developer.apple.com/support/D-U-N-S/

**個人**でも可能だが、アプリ内に「合同会社カケル」と表示するなら法人登録推奨。

### 承認まで
- 個人: 即日〜2日
- 法人: 1〜2週間（D-U-N-S取得込み）

## 8️⃣ App Review 対策・よくある審査落ち理由

### ❌ NG（審査で必ず指摘される）

| NG事項 | 回避策 |
|-------|------|
| 「購入を復元」ボタンがない | ✅ 実装済み（Paywall画面） |
| サブスク解約方法が不明瞭 | ✅ 「iOS設定から解約可能」を明記 |
| プライバシーポリシー欠如 | ✅ URL 登録 |
| 課金後に「7日間無料」の明示なし | ✅ 登録完了時メッセージで明示済み |
| アプリ内でウェブ決済への誘導 | ⚠️ iOSビルドでは Apple IAP 必須・PayPal UIを非表示に |
| アカウント削除機能なし | ✅ 実装済み |
| ダミーデータで機能不明瞭 | 問題データが十分ある ✅ |
| 国際化問題（英語でも動くか） | ✅ 日本語メインで問題なし |

### ⚠️ iOS ビルド時の特別対応

Apple の規約により、**iOS アプリ内でサブスクを販売する場合、Apple IAP 必須**。
PayPal の決済ボタンは iOS では非表示にするか、Apple IAP ボタンに切り替え必要。

※ コード側で `Platform.OS === 'ios'` 分岐して制御する。

## 9️⃣ EAS Build iOS コマンド

```bash
# iOS 本番ビルド
eas build --platform ios --profile production

# 初回は Apple Developer 認証必要
# EAS が自動で証明書を生成・管理する
```

完了後、EAS ダッシュボードから .ipa をダウンロード、または直接 App Store Connect に送信可能:

```bash
# TestFlight にアップロード
eas submit --platform ios --profile production
```

## 💡 戦略的アドバイス

### 登録時期
Apple Developer は $99/年。**実際にiOS版リリースが見えてから登録**しても遅くない。

### 優先順位（おすすめ順）
1. **Web版で販売開始**（PayPal・ローンチ済み）
2. **Google Play**（$25一回・3ヶ月以内）
3. **App Store**（$99/年・Google Playで成果が出たら）

理由: Android 版が先に出た方がリスク低く学べるため（審査・IAP・バイリング）。

### 並行戦略の強み
各プラットフォームで **「同じアカウントでクラウド同期」** 可能。
- Web で無料登録 → PayPal 課金
- Android に後からログイン → 進捗同期
- iOS に後からログイン → 進捗同期

この「デバイス横断」こそ、宅建アプリ市場で他社と差別化できる武器。
